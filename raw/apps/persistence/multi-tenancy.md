# Multi-Tenancy

> Multi-tenant data isolation in Kinotic with automatic tenant filtering.

Kinotic supports multi-tenant data isolation at the persistence layer. Data from different tenants is stored together but automatically filtered so each tenant only sees their own data.

## Defining a Multi-Tenant Entity

Add `@TenantId` to a field and set the entity to `MultiTenancyType.SHARED`.

```typescript
import { Entity, AutoGeneratedId, TenantId, MultiTenancyType } from '@kinotic-ai/persistence'

@Entity(MultiTenancyType.SHARED)
export class Person {
    @AutoGeneratedId
    id: string | null = null

    @TenantId
    tenantId: string = ''

    firstName: string = ''
    lastName: string = ''
}
```

The `@TenantId` field is populated automatically based on the authenticated user's tenant context. You do not need to set it manually when saving entities.

## Tenant-Isolated Service

The standard generated `EntityService` automatically filters all operations to the current tenant. No additional configuration is needed.

```typescript
const service = new PersonEntityService()

// Only returns people belonging to the current tenant
const people = await service.findAll({ page: 0, size: 10 })
```

All CRUD operations (save, find, update, delete, search, count) are scoped to the current tenant transparently.

## Admin Entity Service

When you run `kinotic sync` on an entity with a `@TenantId` field, the CLI generates both a tenant-scoped service (`PersonEntityService`) and an admin service (`PersonAdminEntityService`) that can access data across tenants. The admin service is useful for administrative dashboards, reporting, and cross-tenant operations.

```typescript
const adminService = new PersonAdminEntityService()
```

### Access All Tenants

```typescript
const allPeople = await adminService.findAll(['*'], { page: 0, size: 10 })
```

### Access a Specific Tenant

```typescript
const tenantPeople = await adminService.findAll(['tenant-123'], { page: 0, size: 10 })
```

### Access Multiple Tenants

```typescript
const multiPeople = await adminService.findAll(['tenant-123', 'tenant-456'], { page: 0, size: 10 })
```

## Security Considerations

Admin entity services should be restricted to users with appropriate administrative permissions. The standard tenant-isolated service is the default and should be used for regular application logic.
